### CON-ARCH-001 — X3 interlocked placement clearance for 3D-printed housing

| Field | Value |
| --- | --- |
| Type | constraint |
| Target | kc2-x3 |
| Status | verified |
| Priority | high |
| Tags | kc2 x3 mechanical housing magnet |
| Risk | high |
| Stability | stable |
| Verification Method | Board-coordinate render and kc2-pcb-preflight regression check. |
| GitHub Issue | - |
| Related Docs | docs/spec.md, docs/spec/20.kc2-no-stabilizer-layout.md |

#### Requirement

KC2 X3 shall define an interlocked joined-placement clearance between the left and right PCB Edge.Cuts so the right-side `Y`/`H` protrusion can enter the left-side `T`/`G` recess while the halves can still be used with a 3D-printed bottom housing and magnetic alignment without physical binding. The right-side `Y`/`H` protrusion may locally relieve its horizontal ledges by 0.8 mm while preserving the vertical protrusion face at the 3.6 mm X3 inner-edge margin.

#### Rationale

3D-printed bottom housings can swell or print oversized, and magnets add alignment tolerance. A zero-clearance joined reference is too tight for physical assembly, but a pure bounding-box gap wastes the available T/G and Y/H interlock geometry.

#### Acceptance Criteria

- [x] AC-1: Joined visual and mechanical reference render uses interlocked placement with 1.0 mm minimum Edge.Cuts clearance between the left and right PCBs.
- [x] AC-2: Specification allows 0.8-1.2 mm practical interlocked joined-placement clearance and treats 1.0 mm as the default.
- [x] AC-3: PCB electrical clearance and routing safety are not reduced below the X3 fabrication gate by this placement-clearance change, including the local right `Y`/`H` horizontal ledge relief.

### OPS-ARCH-002 — X3 main KiCad output promotion

| Field | Value |
| --- | --- |
| Type | operational |
| Target | kc2-x3 |
| Status | verified |
| Priority | high |
| Tags | kc2 x3 main promotion draft |
| Risk | high |
| Stability | stable |
| Verification Method | Directory inspection, KiCad CLI DRC, render/verifier scripts, kc2-pcb-preflight, and sub-agent review. |
| GitHub Issue | - |
| Related Docs | docs/spec.md, docs/spec/20.kc2-no-stabilizer-layout.md |

#### Requirement

KC2 shall promote the verified X3 left and right KiCad PCB outputs to the main hardware/kicad/kc2_left and hardware/kicad/kc2_right project paths, while preserving earlier soldered, hotswap, x1, and x2 variants under hardware/kicad/draft for historical reference.

#### Rationale

X3 is the verified no-stabilizer hardware direction. Keeping it at -x3 paths while older variants occupy main paths makes fabrication and tool entrypoints ambiguous.

#### Acceptance Criteria

- [x] AC-1: Existing non-X3 KiCad PCB project directories and variant manifests are moved under hardware/kicad/draft without being deleted.
- [x] AC-2: Verified X3 left and right KiCad projects are available at hardware/kicad/kc2_left and hardware/kicad/kc2_right as the main PCB outputs.
- [x] AC-3: Tooling, renders, fabrication outputs, and documentation reference the promoted main paths or explicitly identify draft paths.
- [x] AC-4: Connectivity, interlock render annotation, KiCad DRC, and kc2-pcb-preflight checks pass after the promotion.
