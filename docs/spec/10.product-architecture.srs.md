### CON-ARCH-001 — X3 interlocked placement clearance for 3D-printed housing

| Field | Value |
| --- | --- |
| Type | constraint |
| Target | kc2-x3 |
| Status | verified |
| Priority | high |
| Tags | kc2 x3 mechanical housing magnet |
| Risk | high |
| Stability | stable |
| Verification Method | Board-coordinate render and kc2-pcb-preflight regression check. |
| GitHub Issue | - |
| Related Docs | docs/spec.md, docs/spec/20.kc2-no-stabilizer-layout.md |

#### Requirement

KC2 X3 shall define an interlocked joined-placement clearance between the left and right PCB Edge.Cuts so the right-side `Y`/`H` protrusion can enter the left-side `T`/`G` recess while the halves can still be used with a 3D-printed bottom housing and magnetic alignment without physical binding. The right-side `Y`/`H` protrusion may locally relieve its horizontal ledges by 0.8 mm while preserving the vertical protrusion face at the 3.6 mm X3 inner-edge margin.

#### Rationale

3D-printed bottom housings can swell or print oversized, and magnets add alignment tolerance. A zero-clearance joined reference is too tight for physical assembly, but a pure bounding-box gap wastes the available T/G and Y/H interlock geometry.

#### Acceptance Criteria

- [x] AC-1: Joined visual and mechanical reference render uses interlocked placement with 1.0 mm minimum Edge.Cuts clearance between the left and right PCBs.
- [x] AC-2: Specification allows 0.8-1.2 mm practical interlocked joined-placement clearance and treats 1.0 mm as the default.
- [x] AC-3: PCB electrical clearance and routing safety are not reduced below the X3 fabrication gate by this placement-clearance change, including the local right `Y`/`H` horizontal ledge relief.

### CON-ARCH-002 — X3 compact controller tab without PCB battery pads

| Field | Value |
| --- | --- |
| Type | constraint |
| Target | kc2-x3 |
| Status | verified |
| Priority | high |
| Tags | kc2 x3 controller compact battery reset |
| Risk | high |
| Stability | stable |
| Verification Method | KiCad Python geometry/power verifier, connectivity verifier, KiCad CLI DRC, kc2-pcb-preflight, fabrication ZIP manifest inspection, and sub-agent review. |
| GitHub Issue | - |
| Related Docs | docs/spec.md, docs/spec/20.kc2-no-stabilizer-layout.md |

#### Requirement

KC2 X3 shall remove the carrier-PCB battery/direct-solder power pad footprint from both halves, route no battery power nets on the PCB carrier, shrink each controller protrusion tab to fit tightly around the nice!nano v2 socket, antenna keepout, USB-C access, and reset tact switch, keep the reset tact switch on the antenna-side controller-tab edge where it remains pressable without entering the nice!nano socket or TW301525 battery reference area, and provide a non-plated battery-lead pass-through slot below the nice!nano v2 B+/B- top-pin area for an optional bottom-side cable exit.

#### Rationale

The previous PCB placed four exposed battery hand-solder pads beside the nice!nano socket, which widened the controller protrusion. Removing those carrier pads allows a tighter controller tab, but it changes the power assembly assumption: the Li-Po leads must be soldered directly to the nice!nano v2 B+/B- pads off the carrier PCB, so the carrier PCB must not contain stale battery nets or exposed power pads.

#### Acceptance Criteria

- [x] AC-1: Both left and right main KiCad boards contain no J_PWR1 footprint, no BAT+/BAT- carrier nets, and no NN_B+/NN_B- carrier pads or traces.

#### Requirement

KC2 X3 shall remove all M2 screw mounting holes from both PCB halves, use 1.6 mm PCB thickness, support an FDM PLA+ lower housing that primarily captures the PCB by printed ledges/rails rather than adhesive, add nine evenly distributed 3.0 mm NPTH non-screw registration holes per half between key switch areas, shift the middle support holes toward the anti-flex center zones, visibly label the support holes as H1-H9, and retreat the PCB Edge.Cuts as far as practical while preserving enough rail land for the printed housing.

#### Rationale

The user wants the X3 keyboard to become thinner and visually tighter than the M2 screw-fixed version. Screw holes and wide perimeter land should be removed where possible, but the PCB still needs repeatable registration and mechanical capture in a 3D-printed PLA+ lower tray.

#### Acceptance Criteria

- [x] AC-1: Both left and right main KiCad boards contain zero M2_NPTH_2.2 mounting-hole footprints and no screw-mount drill features.
- [x] AC-2: Both halves contain exactly nine 3.0 mm NPTH non-screw registration holes, evenly distributed in a 3 x 3 pattern between switch areas, with the middle support row shifted toward the board anti-flex center zones, with visible H1-H9 labels, and with no copper, trace, via, diode, switch, controller, reset, antenna keepout, or Edge.Cuts collision.
- [x] AC-3: The generator documents and applies a 4.0 mm nominal outer rail land, allows local 3.6 mm hard lower bound only where verified safe, and allows up to 5.0 mm where routing, print strength, or tolerance requires it.
- [x] AC-4: Left and right Edge.Cuts are retreated from the previous M2-oriented outline wherever rail land, routing clearance, antenna keepout, USB/reset access, and interlocked joined clearance permit.
- [x] AC-5: Connectivity, KiCad DRC, registration-hole verifier, compact-controller verifier, antenna keepout verifier, joined render annotation verifier, kc2-pcb-preflight, Gerber/Excellon/JLCPCB ZIP verification, and sub-agent review pass before the screwless board is treated as orderable.
