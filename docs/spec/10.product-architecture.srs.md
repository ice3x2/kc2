### CON-ARCH-001 — X3 interlocked placement clearance for 3D-printed housing

| Field | Value |
| --- | --- |
| Type | constraint |
| Target | kc2-x3 |
| Status | verified |
| Priority | high |
| Tags | kc2 x3 mechanical housing magnet |
| Risk | high |
| Stability | stable |
| Verification Method | Board-coordinate render and kc2-pcb-preflight regression check. |
| GitHub Issue | - |
| Related Docs | docs/spec.md, docs/spec/20.kc2-no-stabilizer-layout.md |

#### Requirement

KC2 X3 shall define an interlocked joined-placement clearance between the left and right PCB Edge.Cuts so the right-side `Y`/`H` protrusion can enter the left-side `T`/`G` recess while the halves can still be used with a 3D-printed bottom housing and magnetic alignment without physical binding. The right-side `Y`/`H` protrusion may locally relieve its horizontal ledges by 0.8 mm while preserving the vertical protrusion face at the 3.6 mm X3 inner-edge margin.

#### Rationale

3D-printed bottom housings can swell or print oversized, and magnets add alignment tolerance. A zero-clearance joined reference is too tight for physical assembly, but a pure bounding-box gap wastes the available T/G and Y/H interlock geometry.

#### Acceptance Criteria

- [x] AC-1: Joined visual and mechanical reference render uses interlocked placement with 1.0 mm minimum Edge.Cuts clearance between the left and right PCBs.
- [x] AC-2: Specification allows 0.8-1.2 mm practical interlocked joined-placement clearance and treats 1.0 mm as the default.
- [x] AC-3: PCB electrical clearance and routing safety are not reduced below the X3 fabrication gate by this placement-clearance change, including the local right `Y`/`H` horizontal ledge relief.

### CON-ARCH-002 — X3 compact controller tab without PCB battery pads

| Field | Value |
| --- | --- |
| Type | constraint |
| Target | kc2-x3 |
| Status | verified |
| Priority | high |
| Tags | kc2 x3 controller compact battery reset |
| Risk | high |
| Stability | stable |
| Verification Method | KiCad Python geometry/power verifier, connectivity verifier, KiCad CLI DRC, kc2-pcb-preflight, fabrication ZIP manifest inspection, and sub-agent review. |
| GitHub Issue | - |
| Related Docs | docs/spec.md, docs/spec/20.kc2-no-stabilizer-layout.md |

#### Requirement

KC2 X3 shall remove the carrier-PCB battery/direct-solder power pad footprint from both halves, route no battery power nets on the PCB carrier, shrink each controller protrusion tab to fit tightly around the nice!nano v2 socket, antenna keepout, USB-C access, and reset tact switch, and keep the reset tact switch on the USB-C key-side where it remains pressable without battery interference.

#### Rationale

The previous PCB placed four exposed battery hand-solder pads beside the nice!nano socket, which widened the controller protrusion. Removing those carrier pads allows a tighter controller tab, but it changes the power assembly assumption: the Li-Po leads must be soldered directly to the nice!nano v2 B+/B- pads off the carrier PCB, so the carrier PCB must not contain stale battery nets or exposed power pads.

#### Acceptance Criteria

- [x] AC-1: Both left and right main KiCad boards contain no J_PWR1 footprint, no BAT+/BAT- carrier nets, and no NN_B+/NN_B- carrier pads or traces.
- [x] AC-2: Each controller protrusion tab is reduced from the previous 72.0 mm width while preserving nice!nano socket clearance, antenna no-copper keepout, USB-C access, and board-edge copper clearance.
- [x] AC-3: SW_RST1 remains connected exactly between RST and GND, is placed on the USB-C key-side below the controller, and does not overlap the nice!nano socket, antenna keepout, battery reference area, or Edge.Cuts.
- [x] AC-4: Connectivity, KiCad DRC, compact-controller geometry verification, kc2-pcb-preflight or an explicit preflight compatibility review, fabrication export, JLCPCB ZIP inspection, and sub-agent review pass before the compact board is treated as orderable.
