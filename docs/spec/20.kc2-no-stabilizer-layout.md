## 1. Purpose

This document is the KC2 physical layout subdocument for the plateless low-profile no-stabilizer direction.

The design intentionally splits every current `>=2U` key into smaller physical keys. Duplicate legends such as `LShift`/`LShift`, `BSPC`/`BSPC`, `Enter`/`Enter`, and `Space`/`Space` are accepted as the final physical-key naming decision for this layout.

## 2. Policy

- The no-stabilizer layout uses no `>=2U` physical keys.
- `1.75U` remains the largest physical key width in this layout.
- Existing row extents are preserved so the PCB outline strategy can remain close to the current KC2 draft.
- Firmware may map duplicate physical legends to identical or separate behaviors; the physical layout intentionally does not force that choice.

## 5. Split Decisions

| Original key | Original width | Replacement |
|---|---:|---|
| Left `Shift` | `2.25U` | `1.25U LShift` + `1U LShift` |
| Left `Space` | `2.25U` | `1U Space` + `1.25U Space` |
| Right `BSPC` | `2.25U` | `1U BSPC` + `1.25U BSPC` |
| Right `Enter` | `2.5U` | `1U Enter` + `1.5U Enter` |
| Right `RShift` | `2.0U` | `1U RShift` + `1U Fn` |
| Right `Space` | `2.0U` | `1U Space` + `1U Space` |

## 6. Implementation Notes

- The PCB generator must add 6 physical switch positions and 6 diodes compared with the 71-key routed draft.
- The provisional stabilizer marker generation for `key.w_u >= 2.0` should produce no stabilizer markers for this layout.
- Right-side matrix column assignment must be reviewed. A naive row-array edit can shift column semantics and invalidate the current `R_COL7`/`R_COL8` rationale.
- The ZMK matrix and keymap should explicitly distinguish duplicate physical keys with stable physical IDs even when legends match.
- PCB regeneration, routing, and DRC must be rerun after implementation.
