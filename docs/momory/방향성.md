# KC2 X3 초슬림 무나사 하우징 방향성

작성일: 2026-06-07

관련 Requirement ID: `CON-ARCH-001`, `CON-ARCH-002`, `CON-ARCH-003`, `OPS-ARCH-001`, `OPS-ARCH-002`

## 1. 배경

KC2 X3는 기존 M2 나사 고정 방향에서 벗어나, 더 얇고 외곽이 작은 키보드를 목표로 한다. 다음 KiCad 구현 전에 SRS를 갱신해야 하며, 이 문서는 현재 결정 방향을 기록하는 설계 메모다.

## 2. 고정 방식 결정

- 기존 M2 나사 구멍은 모두 제거한다.
- PCB 두께는 1.6 mm를 사용한다.
- 하우징은 FDM 방식 PLA+ 출력물을 전제로 한다.
- 1차 고정은 접착제가 아니라 3D 프린터 하우징에 PCB가 걸리는 구조로 한다.
- 접착제는 필요할 경우 보조 수단으로만 검토한다.
- 하우징은 얕은 트레이, 내부 턱, 레일, 또는 이 조합으로 PCB의 Z축 이탈과 XY 흔들림을 막는다.

## 3. 비나사 등록 구멍 결정

5명 설계 위원회 결과, 비나사 등록 구멍 지름과 개수는 5/5 만장일치로 다음과 같이 결정한다.

| 항목 | 결정 |
| --- | --- |
| 구멍 성격 | 나사 체결용이 아닌 하우징 등록/보조 포획용 NPTH |
| 구멍 지름 | 3.0 mm |
| 구멍 수 | 각 half당 9개, 총 18개 |
| 하우징 peg 권장 지름 | 2.6-2.8 mm |
| copper/trace/via clearance | hole edge 기준 목표 1.0 mm 이상 |
| 최저 허용 clearance | DRC와 preflight가 통과하는 범위에서만 예외 검토 |

2.0 mm는 FDM PLA+ peg 강도와 출력 공차에 불리하고, 4.0 mm는 19.05 mm 키 피치 안에서 FR4 web, 스위치 패드, 다이오드, 라우팅을 과하게 침범할 가능성이 크다. 따라서 3.0 mm NPTH를 기본값으로 한다.

## 4. 등록 구멍 위치 정책

등록 구멍 9개는 각 half 안에서 3 x 3으로 고르게 분포시킨다. 기존 외곽 M2 위치를 재사용하지 않고, 버튼 사이 내부 빈 공간에서 새로 고른다.

위치 산출 원칙:

- 후보는 인접 키 row 사이 y-midline과 인접 키 column gap의 x-midline을 기준으로 만든다.
- 가능하면 네 개의 스위치가 둘러싼 빈 중심을 우선한다.
- 상/중/하 3개 band와 좌/중/우 3개 zone을 고르게 채운다.
- left와 right는 독립 PCB이므로 각각 9개를 배치한다.

회피해야 할 영역:

- switch through-hole pad, hotswap pad, socket courtyard
- SOD-123 diode pad/body/courtyard
- matrix trace, via, controller routing bus
- nice!nano v2 socket, antenna keepout, USB-C 접근부
- `SW_RST1`
- 결합 edge의 얇은 interlock 구간
- Edge.Cuts와 너무 가까운 영역
- 하우징 레일이 직접 물어야 하는 외곽 land

현재 보드 좌표계에서 시작 후보로 사용할 y band:

| Band | 후보 y |
| --- | ---: |
| row0-row1 gap | 87.05 mm |
| row1-row2 gap | 106.10 mm |
| row2-row3 gap | 125.15 mm |
| row3-row4 gap | 144.20 mm |

구현 시에는 이 후보 band 중 3개를 선택하되, 실제 pad/trace/diode 충돌 검사 결과에 따라 교체한다. 최종 좌표는 KiCad 보드에 3.0 mm NPTH와 copper keepout을 넣은 뒤 DRC, connectivity, antenna keepout, fabrication ZIP 검증을 통과해야 확정한다.

## 5. 외곽선 후퇴와 레일 여유

외곽선은 가능한 한 모두 후퇴시킨다. 단, 3D 프린터 하우징이 PCB를 걸 수 있는 land는 남긴다.

5명 설계 위원회 의견은 3.0 mm, 4.0 mm, 4.5 mm, 5.0 mm로 갈렸다. 다수/중앙값인 4.0 mm를 nominal 값으로 삼고, 최종 방향은 다음과 같이 잡는다.

| 항목 | 결정 |
| --- | --- |
| 일반 외곽 rail land | 4.0 mm nominal |
| 국소 hard lower bound | 3.6 mm |
| 강도/공차/배선 때문에 필요한 구간 | 5.0 mm까지 허용 |
| 결합 inner edge | 기존 X3 interlock clearance와 별도 관리 |

4.0 mm nominal은 초슬림 목표와 FDM PLA+ 조립 공차의 균형값이다. 3.0 mm는 매우 공격적인 값이라 마모, 출력 편차, copper-edge clearance에 취약할 수 있고, 5.0 mm는 안정적이지만 외곽 후퇴 목표와 충돌한다.

하우징 설계 가정:

- PCB slot 높이는 1.6 mm PCB를 기준으로 실제 출력 후 조립 여유를 둔다.
- rail/lip은 PCB를 누르는 구조보다 PCB가 옆에서 들어가 걸리는 구조를 우선 검토한다.
- 결합 edge의 얇은 interlock 구간은 강한 레일 포획보다 하부 연속 rib로 받치는 방식을 우선한다.

## 6. 구현 게이트

다음 단계에서 KiCad 보드를 수정하기 전에 SRS를 갱신해야 한다. 현재 SRS와 검증 증거는 M2 기반 고정 이력을 포함하고 있으므로, 무나사 레일/등록홀 구조는 별도 요구사항 또는 기존 요구사항 변경으로 추적되어야 한다.

구현 시 필수 갱신 항목:

- `tools/generate_kc2_pcbs.py`: M2 mounting hole 제거, 3.0 mm NPTH 등록홀 생성, 외곽 margin 후퇴
- KiCad 좌/우 main board
- DRC JSON
- connectivity, antenna keepout, compact controller, render annotation verifier
- KiCad 실행 없는 정적 텍스트 verifier
- 등록홀 전용 verifier
- render PNG/SVG
- Gerber/Excellon/JLCPCB ZIP
- `kc2-pcb-preflight`
- sub-agent 검증

완료 판단은 자동 DRC만으로 하지 않는다. 실제 pad net, trace clearance, antenna keepout, controller/USB/reset, fabrication ZIP drill/mask 데이터를 모두 확인해야 한다.
